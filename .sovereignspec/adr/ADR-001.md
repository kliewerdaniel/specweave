# ADR-001: 7-Layer Architecture with Local-First Design

## Context
SpecWeave requires a modular, local-first architecture that can evolve independently across seven functional layers. Each layer has distinct technology needs: inference (Ollama), persistence (SQLite/ChromaDB/NetworkX), reasoning (speculative + neuro-symbolic), compilation (self-verifying pipeline), agent gateway (A2A/MCP), and interface (Next.js dashboard).

## Decision
Adopt a strict 7-layer architecture where each layer only depends on layers below it:

1. **Local Inference** — Ollama/llama.cpp, no cloud APIs
2. **Persistence** — SQLite (metadata), ChromaDB (vectors), NetworkX (graph)
3. **Speculative Reasoning** — draft-verify-commit pipeline over LLM
4. **Neuro-Symbolic KG** — symbolic validation over neural generation
5. **Self-Verifying Compiler** — 6-gate sequential pipeline
6. **A2A/MCP Agent Gateway** — agent federation on localhost
7. **Interface** — Next.js dashboard (separate from backend)

## Rationale
- Layer isolation prevents cascading failures
- Each layer can be tested independently
- New layers can be added without modifying existing ones
- Enables independent scaling of inference, storage, and API tiers

## Alternatives Considered
- **Monolith**: Simpler but would couple inference to API, making offline/online splits hard
- **Microservices with gRPC**: Overkill for single-machine deployment; HTTP/A2A is sufficient
- **Event-driven**: Adds complexity without benefit for local-first use case

## Consequences
- Positive: Clear dependency direction, independent testing, parallel development
- Positive: Each layer can be replaced independently (e.g., swap ChromaDB for FAISS)
- Negative: Cross-layer data must pass through defined interfaces
- Negative: Interface layer (Next.js) requires separate setup from backend
