# ADR-003: Neuro-Symbolic Hybrid Loop for Spec Verification

## Context
Spec verification requires both deterministic correctness (field presence, dependency cycles) and semantic understanding (contradictions, intent alignment). Pure neural approaches are non-deterministic; pure symbolic cannot understand semantics.

## Decision
Implement a hybrid neuro-symbolic loop:
- **Neural generates** — LLM produces architecture descriptions, spec sections, and candidates
- **Symbolic validates** — NetworkX performs deterministic checks (cycles, completeness, constraint satisfaction)
- **Neural revises on failure** — if symbolic validation fails, the neural layer is prompted to revise the failing section with the failure reason as context

## Rationale
- Combines LLM flexibility with deterministic guarantees
- Failed symbolic checks provide structured feedback for neural revision
- All constraint queries are deterministic with no AI inference (auditable)
- ChromaDB provides semantic consistency checks across the corpus

## Alternatives Considered
- **Pure neural**: Non-deterministic, cannot guarantee spec correctness
- **Pure symbolic**: Cannot understand semantics, contradictions, or intent alignment
- **Human-in-the-loop**: Too slow for automated compilation pipeline

## Consequences
- Positive: High confidence in spec correctness
- Positive: Neural revision learns from symbolic failures
- Negative: Two-stage process (neural + symbolic) adds latency
- Negative: Requires both Ollama and NetworkX/ChromaDB to be available
